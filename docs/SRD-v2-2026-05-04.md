## 0. 元信息与仲裁链

### 0.1 文档定位

- PRD 负责“产品是什么、边界是什么、何时解锁”。
- SRD 负责“当前工程如何实现、如何验证、如何不越界”。
- specs 负责更窄的 contract 明细。

### 0.2 当前仲裁链

```text
user 最新口头
  > docs/current.md / docs/task-index.md / docs/decision-log.md
  > PRD v2 / SRD v2
  > specs contract files
  > archive / historical notes
  > chat summary
```

### 0.3 单 user / local-first 假设

- 不做多用户、auth、RBAC、SaaS、容器编排。
- 默认本机 `127.0.0.1`，SQLite + FS 双事实源。
- 任何 runtime 扩张都要走单独 task gate。

## 2. 当前功能基线（FR）

### 2.1 Capture entry

当前有效入口：

- `platform=bilibili`
- `source_kind=manual_url`
- `quick_capture_preset=metadata_only`

必须拒绝：

- recommendation / keyword / RAW gap 直接创建 capture
- `audio_transcript`
- XHS / YouTube runtime capture
- 未授权扩 scope

### 2.2 Receipt / artifact ledger

- `POST /jobs/{job_id}/complete` 是 API-side receipt ingestion 和 ledger entrypoint。
- receipt 只接收当前 phase 允许的 artifact kinds。
- `artifact_assets` 是统一 authority 命名。
- generic metadata-fetch receipt 不得把 blocked/no-auth evidence 假装成 success evidence。

### 2.3 Trust trace

- `GET /captures/{capture_id}/trust-trace` 必须分层返回：
  - `capture`
  - `capture_state`
  - `metadata_job`
  - `probe_evidence`
  - `receipt_ledger`
  - `media_audio`
  - `audit`
- trust trace 只显示 safe summary，不提升 runtime approval。

### 2.4 Metadata probe evidence path

当前已证明的安全路径：

```text
tool preflight
  -> auth-present metadata probe
  -> redacted evidence
  -> metadata probe receipt bridge
  -> trust trace
```

当前未证明也未批准的路径：

- no-auth live runtime beyond bounded probe
- media download
- ffmpeg
- ASR
- audio transcript

### 2.5 Manual auth gate

- QR / manual auth 是独立高风险 gate。
- repo 内不得留下 `BBDown.data`、`BBDownTV.data`、`qrcode.png`、cookie、token、signed URL。
- manual auth 如果执行，必须在 repo 外 local-only executable/store/cwd 上进行。

### 2.6 Runtime gate split

- `ToolPreflightResult` 负责本地工具可用性。
- `PlatformResult` 负责平台边界结果。
- `executable_not_found` 不得进入 `PlatformResult` enum。
- 当前 `C-PLT-001` 的 12 enum 保持不动。

### 2.7 Phase 2 entity outline

`signals / hypotheses / capture_plans / topic_cards` 在 v2 中保留为稳定工程轮廓：

- 它们属于未来 FR 设计方向
- 不等于当前 task 已批准代码实现
- 后续实现必须从 v2 开始，而不是回到 v1 amendment 链

## 3. 数据与状态（DR）

### 3.1 Authority 三件套

- SQLite：结构化数据、状态、关系
- FS：大对象与证据寻址
- state words：跨 worker / 跨 agent 的语义契约

### 3.2 当前必须保持的状态语义

- capture lifecycle 仍是现有工程主链
- job status、artifact ledger、trust trace 标签必须与 current phase 一致
- `audio_transcript` 在状态和 UI 口径里都必须保持 blocked

### 3.3 不可随意改动的 contract 本体

以下 spec 在本 promote 中只作为引用，不允许被改写语义：

- `docs/specs/raw-response-redaction.md`
- `docs/specs/worker-receipt-contract.md`
- `docs/specs/platform-adapter-risk-contract.md`
- `docs/specs/bbdown-adapter-contract-draft.md` 仍是 draft

### 3.4 命名与路径

- `artifact_assets` 为当前权威命名
- `capture creation entrypoint` 为 `/captures/discover` 当前语义
- `data/`、`referencerepo/` 仍是 local-only / git 禁区

## 4. 接口需求（IF）

### 4.1 API 层职责

API 只负责：

- validation
- idempotency
- state transition enforcement
- receipt ingestion
- safe trace projection

API 不负责：

- 业务策略本身
- 长任务执行
- LLM prompt 逻辑

### 4.2 Worker ↔ API 边界

- Worker 不能直接连 SQLite
- Worker 不能绕开 API 直写 authority
- Worker 可按约定写 FS，但 durable registration 仍通过 API 回报

### 4.3 Console ↔ API 边界

- Console 无直写 authority 权限
- Console 失败 / 不打开不应阻断底层基线
- frontend mode 当前不在 active lane

## 5. 非功能需求（NFR）

### 5.1 当前保留的硬 NFR

- 本机 API P95：
  - `/captures/discover` < 500ms
  - `/jobs/{id}/complete` < 1s
  - `/captures/{id}/trust-trace` < 500ms
- 核心 state guard / redaction / receipt / PlatformResult / LP-001 覆盖率 >= 95%
- API DTO / Pydantic / storage / adapter result 边界 100% type hint
- `raw_api_response` evidence 必须 `redaction_applied=true`

### 5.2 已降级或 deferred 的 NFR

- frontend 可用性指标
- dashboard freshness
- token/cost 入库
- test duration NFR
- docstring 强制
- TypeScript strict（frontend lane 才启）

这些已在 `T-P1A-014` 减重，不得在 v2 promote 时偷偷恢复。

## 6. 系统约束（CR）

### 6.1 运行与权限约束

- 默认 bind `127.0.0.1`
- 单 user 假设
- Phase N 未通过，Phase N+1 不开工
- `audio_transcript` blocked

### 6.2 ToS / 安全约束

- 尊重平台访问边界与低频原则
- 凭据不进 Git、PR、CI、logs、DB artifacts
- local tool preflight failure 不得伪装成 platform failure

### 6.3 并行执行约束

- `Active product lane max = 3`
- `Authority writer max = 1`
- `Same file group writer max = 1`
- review / audit / research lane 不计入 product lane，除非写 authority
