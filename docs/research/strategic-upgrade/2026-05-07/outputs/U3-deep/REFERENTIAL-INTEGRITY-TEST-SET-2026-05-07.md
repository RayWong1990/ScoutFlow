## §2 Test groups and rationale

[candidate] Signal tests RI-001..RI-005: ensure observation primitives do not become a second capture inbox and preserve unknown metadata honesty.
[candidate] Hypothesis tests RI-006..RI-010: ensure claim nodes cannot outrun evidence or user verdict gates.
[candidate] CapturePlan tests RI-011..RI-015: ensure future orchestration remains blocked unless LP-001 and runtime gates are explicit.
[candidate] TopicCard tests RI-016..RI-020: ensure lite carry-over fields and v1 relation fields coexist without RAW/DiloFlow overclaim.
[candidate] Cross-system tests RI-021..RI-027: ensure RAW, DiloFlow, and multiple local workflows do not create dual-SoR ambiguity.
[candidate] Meta-contract tests RI-028..RI-036: ensure candidate flags, live-web honesty, ULID/id conventions, schema pins, and anti-second-inbox rules are machine-auditable.

## §3 Minimal pseudo-runner

```python
# [candidate] fixture-level pseudo-runner; not production code.
def validate_bundle(bundle):
    errors = []
    for signal in bundle.signals:
        if not bundle.has_capture(signal.source_capture_id):
            signal.lifecycle_state = "broken_reference"
        if signal.copies_capture_state_directly:
            errors.append(("RI-032", signal.id, "anti_second_inbox_fail"))
    for hypothesis in bundle.hypotheses:
        missing = [sid for sid in hypothesis.support_signal_ids if not bundle.has_signal(sid)]
        if missing:
            hypothesis.lifecycle_state = "needs_repair"
        if hypothesis.user_verdict == "reject":
            for plan in bundle.plans_for_hypothesis(hypothesis.id):
                plan.lifecycle_state = "suspended_pending_user_review"
    for plan in bundle.capture_plans:
        if plan.source_kind in {"recommendation", "keyword", "raw_gap"}:
            plan.lifecycle_state = "scope_required"
        if "audio_transcript" in plan.requested_enrichments:
            plan.lifecycle_state = "blocked_runtime"
    for card in bundle.topic_cards:
        if card.raw_handoff.get("true_write") is True:
            errors.append(("RI-019", card.id, "redline_fail"))
    return errors, bundle
```
[candidate] The pseudo-runner expresses the fallback philosophy: record the problem, move affected objects to repair/suspended/broken_reference, and keep evidence available. It intentionally does not delete rows, emit jobs, write RAW, or call DiloFlow.
