## §3 Interpretation rules

[claim]
[instruction] Platform tools are split into local preflight and platform result. A missing executable, failed install, blocked local path, or absent credential is never a platform success/failure claim. The dispatch must preserve ToolPreflightResult / PlatformResult separation.

[boundary]
[instruction] ASR-related names are planning references only. Whisper, faster-whisper, Voxtral, ffmpeg, and transcript normalization cannot be chained into `audio_transcript` without a separate runtime gate, local-only storage plan, redaction contract, and external audit.

[plan]
[instruction] Browser-related names are planning references only. Playwright or browser profiles cannot be used to collect login state, QR image, sidecar, cookie, token, signed URL, or raw screenshot containing sensitive material.

[boundary]
[instruction] Visual and UI references are design vocabulary only. OpenDesign or shadcn-admin patterns may inspire constraints, but this catalog does not add packages, copy layouts, transplant runtime code, or change React/Vite strategy.

[claim]
[instruction] Egress and downstream references are manifest-based. RAW, Obsidian, and DiloFlow receive candidate material through controlled handoff; they do not become ScoutFlow authority and they do not back-write repo truth.

[claim]
## §4 Risk register

[audit] High risk: BBDown, yt-dlp, ffmpeg, ASR models, Playwright, login-required flows, database migrations, and true vault writes. These require explicit future dispatches and can never be opened from this recap.

[audit] Medium risk: hybrid search, visual DAM, agent orchestration, cost ledger, and state library. These can produce docs-only contracts now, but they need external audit before code-bearing work.

[audit] Low risk: vendor table maintenance, citation packaging, README indexing, and self-audit. These still need claim labels because stale vendor evidence can mislead later workers.
